# event-prometheus-processor

This plugin for [gNMIc](https://gnmic.openconfig.net/) can transform event data collected from
devices with gNMI into a format suitable for [Prometheus](https://prometheus.io).

Main features:
  * Rename tags
  * Strip tag and value names to produce concise metric names and labels
  * Convert values to tags
  * Convert timestamps to unixtime
  * Convert enum string values to integers
  * Value allow/block list
  * Create info metrics
  * Adding metadata as tags to access point data from a Cisco WLC

## Downloading

Download prebuilt binaries from [GitHub](https://github.com/adaricorp/gnmic-event-prometheus-processor/releases/latest).

## Configuration

The basic configuration for gNMIc to use this plugin to transform event data before sending it
to a prometheus remote write server, assuming the `event-prometheus-processor`
binary has been installed in `/usr/local/lib/gnmic`, looks like:

```yaml
plugins:
  path: /usr/local/lib/gnmic
  glob: "*"
  start-timeout: 0s

targets:
  wlc.example.org:
    address: wlc.example.org:50052
    insecure: true
    subscriptions:
      - wlc

subscriptions:
  wlc:
    encoding: proto
    stream-subscriptions:
      - paths:
          - ...
        stream-mode: sample
        sample-interval: 60s

processors:
  cisco-wlc:
    event-prometheus-processor:
      name: cisco-wlc
      metric-prefix: cisco
      condition: '.tags.["subscription-name"] == "wlc"'
      cisco-wlcs:
        - wlc.example.org

outputs:
  prometheus_push:
    type: prometheus_write
    url: http://localhost:9090/api/v1/write
    event-processors:
      - cisco-wlc
```

The event-prometheus-processor plugin has many different options for configuring
how events are transformed:

```yaml
event-prometheus-processor:
  # Optional unique name to identify this instance of the plugin
  name: ""
  # JQ expression, the plugin will only modify events for which this expression evaluates to true
  condition: ""
  # Enable debug logging for the plugin
  debug: false
  # Apply a static prefix to all metrics generated by this plugin
  metric-prefix: ""
  # Write all events received by this plugin to the file path configured
  event-trace-file: ""
  # List of target names that should be treated as Cisco WLCs
  cisco-wlcs: []
  # List of tags which must exist for AP metrics
  ap-tags: []
  # List of YANG paths, if an event matches one of these then it will bypass the required tags configured in ApTags
  bypass-ap-tags: []
  # List of YANG paths, if an event matches one of these then it will be converted to a single info metric regardless of the type of the values
  mixed-info-metrics: []
  # List of YANG paths, if an event does not match any of these it will be dropped
  value-allowlist: []
  # List of YANG paths, if an event matches any of these it will be dropped
  value-blocklist: []
  # List of configuration for renaming tags
  tag-renames:
    - # Apply the tag renames to any event that matches this YANG path
      match: ""
      # List of tag renames
      renames:
        - # Original tag name
          from: ""
          # New tag name
          to: ""
  # List of configuration for converting values to tags
  value-tags:
    - # Apply the configuration to any event that matches this YANG path
      match: ""
      # Apply the configuration to any value in this list
      values: []
  # List of configuration for converting enum string values to integers
  enum-mappings:
    - # Apply the enum value mappings to any event that matches this YANG path
      match: ""
      # List of enum value mappings
      enums:
        - # Enum name
          enum: ""
          # List of value mappings
          mappings:
            - # Original string value
              from: ""
              # New integer value
              to: 0
  integer-info-metrics:
    - # Apply the configuration to any event that matches this YANG path
      match: ""
      # Apply the configuration to any value in this list
      values: []
  value-and-info-metrics:
    - # Apply the configuration to any event that matches this YANG path
      match: ""
      # Apply the configuration to any value in this list
      values: []
```
